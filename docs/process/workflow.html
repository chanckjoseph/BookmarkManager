Version Control Strategy (Sprint Branching)

The Release Manager (The Sentinel) enforces a strict branching hierarchy:

# Main Source of Truth
main # Sprint Integration
└── sprint/sprint-01 # Feature Development
└── feat/105-api-prototype
  • Main Branch: Production stable. Merged only at the end of a sprint.
  • Sprint Branches: Active integration for all sprint features.
  • Feature Branches: Task-specific, branching off the active sprint branch.
